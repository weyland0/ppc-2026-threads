## Введение

Численное интегрирование функций многих переменных
является важной задачей вычислительной
математики, возникающей в физике, экономике,
инженерных расчётах и других областях.

Метод Симпсона (парабол) для многомерного случая строится
как тензорное произведение
одномерных формул и обладает повышенной
точностью для гладких функций.

В условиях современного параллельного программирования реализация таких алгоритмов с использованием
различных технологий (OpenMP, TBB, STL) позволяет исследовать эффективность распараллеливания и
выбрать оптимальный подход в зависимости от
аппаратной платформы и размера задачи.

## Постановка задачи

**Цель работы:**  
Реализовать последовательную и параллельные версии
алгоритма вычисления определённого интеграла
функции нескольких переменных по прямоугольной области методом
Симпсона, провести их тестирование и сравнить производительность.

**Определение задачи:**  
Для заданной функции, определённой на прямоугольной области,
требуется вычислить приближённое
значение интеграла с использованием составной формулы Симпсона.
Область разбивается по каждому
измерению на n равных частей (число разбиений должно быть чётным).

Интеграл аппроксимируется взвешенной суммой значений функции в узлах сетки.

**Ограничения:**

- Размерность d >= 1.
- Векторы границ a и b должны иметь одинаковую длину.
- Количество разбиений n должно быть положительным и чётным.
- Функция должна быть определена во всех узлах сетки.

## Описание алгоритма (последовательная версия)

Последовательный алгоритм обходит все узлы многомерной сетки в
лексикографическом порядке
(последнее измерение меняется быстрее всего).

Для каждого узла вычисляется вес Симпсона по каждому измерению:

- w = 1 для граничных узлов i = 0 или i = n_i;
- w = 4 для внутренних узлов с нечётным индексом;
- w = 2 для внутренних узлов с чётным индексом.

Общий вес узла равен произведению одномерных весов. Затем значение
функции в узле умножается на этот вес и суммируется.
После обхода всех узлов результат умножается на
(h_1 *h_2* ... * h_d) / (3^d), где h_i = (b_i - a_i)/n_i.

## Схемы распараллеливания

### OpenMP-версия

В OpenMP используется параллельный цикл по линейному индексу узлов. Предварительно вычисляются
"шаги" (strides) для преобразования линейного индекса в многомерные координаты.

Это позволяет избежать вложенных циклов и обеспечить
равномерное распределение работы между потоками.

#### Принцип работы OpenMP-версии

1. Вычисляются шаги сетки и их произведение.
2. Строится массив strides, где strides[0] = (n_1+1) *
(n_2+1)* ... *(n_{d-1}+1),
strides[1] = (n_2+1)* ... * (n_{d-1}+1), ..., strides[d-1] = 1.
Это позволяет по линейному индексу
idx восстановить индексы по каждому измерению: indices[j] =
(idx / strides[j]) % (n_j+1)
(в коде реализовано через деление и остаток последовательно).
3. Параллельный цикл #pragma omp for schedule(static) с
редукцией по переменной total_sum распределяет итерации между потоками.
4. Каждый поток вычисляет веса и значение функции в своих
узлах, накапливает локальную сумму.
5. После цикла результат усредняется с общим множителем.

### TBB-версия

В TBB используется шаблон parallel_reduce, который автоматически
разбивает диапазон линейных
индексов на блоки и выполняет редукцию.

#### Принцип работы TBB-версии

1. Аналогично вычисляются шаги и произведение.
2. Вызывается tbb::parallel_reduce с диапазоном [0, total_points),
начальным значением 0.0,
функцией обработки блока и функцией объединения.
3. В лямбде-обработчике для каждого индекса в блоке вычисляется
вклад узла (вызов вспомогательной функции ComputeNodeContribution).
4. Результаты блоков суммируются.

### STL-версия

Диапазон индексов разбивается на блоки, размер блока
определяется числом аппаратных потоков.

Для каждого блока запускается асинхронная задача,
результаты собираются через std::future::get().

#### Принцип работы STL-версии

1. Определяется количество потоков num_threads =
std::thread::hardware_concurrency().
2. Вычисляется размер блока и распределяется остаток.
3. Для каждого потока запускается std::async с лямбдой,
вычисляющей сумму на своём диапазоне.
4. Все future собираются, результаты суммируются.
